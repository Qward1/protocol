## D. Калибровка и донастройка (что нашли и исправили)

В ходе тестирования выявлены и устранены 4 дефекта. Все правки — на стороне
backend (границы «LLM-логика в Dify / оркестрация в backend» соблюдены), покрыты
регрессионными тестами.

### D1. Срок из Dify не разбирался (`deadline_at` всегда `NULL`) — КРИТ.
- **Симптом:** workflow по своему же промпту отдаёт `deadline` в формате
  `YYYY-MM-DDTHH:mm:ss`, но `parse_deadline` не имел этого формата →
  `'2026-07-25T00:00:00' -> None`. Ломались напоминания, «просрочено», аналитика по срокам.
- **Фикс:** `app/services/deadlines.py` — добавлен формат `%Y-%m-%dT%H:%M:%S`.
- **Тест:** `test_local_iso_t_datetime_with_seconds`. Проверено в TC07.

### D2. Статус задачи из Dify не нормализовался.
- **Симптом:** статус подставлялся «как есть» (`item.get("status") or "Новое"`);
  произвольное значение LLM могло выпасть из фильтров дашборда.
- **Фикс:** `app/api/protocols.py` — `_normalize_status()` мягко приводит статус к
  `TASK_STATUSES`, неизвестное → «Новое» (по аналогии с `_normalize_priority`).
  Валидные значения workflow («Новое»/«Требует проверки») сохраняются.
- **Тест:** `test_apply_dify_protocol_normalizes_status`. Проверено в TC06.

### D3. Утечка неразрешённого шаблона Dify в справку-обоснование.
- **Симптом:** когда узел «Поиск должностных обязанностей» не подключён к датасету,
  в поле `duty` попадал сырой шаблон `{{#context#}}`.
- **Фикс:** `app/api/tasks.py` — `_strip_unresolved()` вырезает неразрешённые
  плейсхолдеры `{{#…#}}` из `duty/text/fragment`; поле показывается пустым («—»).
- **Тест:** `test_strip_unresolved_dify_placeholder`. Проверено в TC10.

### D4. Классификатор Dify уводил Q&A в ветку извлечения протокола.
- **Симптом:** backend клал в маршрутную строку (`sys.query`) весь контекст встречи;
  классификатор «перетягивал» запрос в извлечение протокола и возвращал
  сгенерированный протокол вместо ответа на вопрос.
- **Фикс:** `app/api/qa.py` — в маршрутную строку кладём только короткий вопрос;
  полный контекст workflow-узел «Вопросы» читает из `inputs.context`
  (`{{#start.context#}}`). Промпты в Dify не менялись.
- **Тест:** `test_qa_keeps_context_out_of_routing_query`. Проверено в TC09 + стабильность 3/3.

## E. Известные ограничения внешнего окружения (не дефекты приложения)

Приложение по всем пунктам деградирует мягко (без 500), как и задумано в архитектуре.

- **E1. Dify Dataset `retrieve` → HTTP 500 на сервере** для всех методов
  (`semantic/keyword/full_text/hybrid`). Индексация документов при этом работает
  (`indexing_status: completed`). Похоже, в инстансе Dify **не переподключена модель
  эмбеддингов** (`bge-m3-v2`, см. `CLAUDE.md`/`dify/WORKFLOW.md`).
  *Поведение приложения:* `/api/search` уходит в локальный полнотекстовый
  (подстрочный) фоллбэк — работает.
  *Рекомендация:* переподключить модель эмбеддингов в датасете Dify — тогда включится
  семантическая «историческая память».

- **E2. Служебная записка (memo).** Плагин `document-generator` в Dify не подключён →
  DOCX из Dify не приходит; backend формирует DOCX локально (штатный фоллбэк, файл в
  `storage/exports`). В веб-режиме файл не отдаётся через API — штатный канал доставки
  справки — **бот MAX** (в этой конфигурации отключён). Генерация подтверждена (TC12).
  *Рекомендация (опц.):* подключить плагин document-generator и/или добавить веб-скачивание memo.

- **E3. Год в сроке.** Если в реплике год не назван, LLM иногда подставляет прошлый
  (наблюдалось `2024` вместо `2026`). *Рекомендация:* добавить текущую дату в системный
  промпт узла извлечения протокола в Dify (граница ответственности — workflow, не backend).

- **E4. ASR/ffmpeg не настроены** в этой конфигурации (по условию задачи). Пайплайн
  распознавания медиа вживую не проверялся; функционал протокол/Q&A/поиск проверен через
  ввод готового текста.
